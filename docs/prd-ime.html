1. Overview del proyecto

IME Calibraciones es un laboratorio de calibración acreditado en Tlaxcala, México. Atiende industrias manufactureras, automotrices, farmacéuticas y laboratorios que requieren calibración certificada de instrumentos de medición. Su nombre legal es Ingeniería y Metrología Certificada Para Calibración de Equipos de Control y Pruebas S.A. de C.V.

El sitio actual es WordPress con plugins, lento, con PageSpeed bajo, secciones incompletas y sin estructura SEO. El cliente ya tiene anticipo pagado para la Fase 1.

Objetivos del cliente (en orden de prioridad)

  1. Visibilidad en Google y motores IA — aparecer en búsquedas como "calibración en Tlaxcala", "laboratorio de calibración acreditado México", y consultas específicas de instrumentos.
  2. Catálogo de productos profesional — comparable o superior a suministrosenmetrologia.com, con filtros y búsqueda.
  3. Carrusel de 32 magnitudes en el home — presentar visualmente el alcance de sus servicios de calibración.

2. Stack & Arquitectura

Stack tecnológico
  • HTML5 semántico (sin framework)
  • CSS3 — variables CSS, flexbox, grid, sin preprocesadores
  • JavaScript ES6+ vanilla — sin React, Vue, ni jQuery
  • JSON como capa de datos para el catálogo
  • GitHub Pages — hosting estático, $0/mes
Por qué este stack
  • Sin costos de hosting ni mantenimiento de servidor
  • Sin base de datos que mantener ni plugins que caducan
  • Carga ultra-rápida → mejor PageSpeed → mejor SEO
  • El cliente recibe el código completo, sin dependencias de terceros
  • 150 productos en JSON = ~80KB, filtrado en memoria = instantáneo

Principios de arquitectura

  • Una página HTML por URL — sin routing SPA
  • CSS custom properties para toda la paleta y tipografía — fácil de mantener
  • Componentes HTML reutilizables via JS include o inline (header, footer, nav)
  • Catálogo: productos.json es la única fuente de verdad. Nunca duplicar datos en HTML.
  • Todas las imágenes optimizadas en WebP + fallback JPG
  • Sin JavaScript en el critical path — CSS carga primero, JS al final o defer

Implementación de filtros (catalog.js)

// Lógica de filtrado — todo en memoria, sin fetch adicional
const state = {
  busqueda: '',
  filtros: {
    magnitud: new Set(),
    marca: new Set(),
    categoria: new Set()
  }
};

function filtrarProductos(productos, state) {
  return productos.filter(p => {
    const matchBusqueda = !state.busqueda ||
      [p.nombre, p.marca, p.modelo, ...(p.tags || [])]
        .join(' ')
        .toLowerCase()
        .includes(state.busqueda.toLowerCase());

    const matchMagnitud = state.filtros.magnitud.size === 0 ||
      state.filtros.magnitud.has(p.magnitud);

    const matchMarca = state.filtros.marca.size === 0 ||
      state.filtros.marca.has(p.marca);

    const matchCategoria = state.filtros.categoria.size === 0 ||
      state.filtros.categoria.has(p.categoria);

    return matchBusqueda && matchMagnitud && matchMarca && matchCategoria;
  });
}

// Los filtros disponibles se generan dinámicamente desde el JSON
// — nunca hardcodear opciones en el HTML

URL con estado de filtros

Los filtros activos se reflejan en la URL como query params para que sean linkables y compartibles:

/catalogo/?magnitud=Temperatura&marca=Fluke
/catalogo/?busqueda=termometro+infrarrojo

Al cargar la página, leer los query params y aplicar los filtros automáticamente.

Generación de páginas de producto

Las 150 páginas HTML se generan con un script Node.js local. No requiere build server en producción — el output es HTML estático.

// scripts/generate-catalog.js
const fs = require('fs');
const path = require('path');

const data = JSON.parse(fs.readFileSync('./data/productos.json'));
const template = fs.readFileSync('./templates/producto.html', 'utf8');

data.productos.forEach(producto => {
  const html = renderTemplate(template, producto);
  const dir = `./catalogo/${producto.slug}/`;
  fs.mkdirSync(dir, { recursive: true });
  fs.writeFileSync(path.join(dir, 'index.html'), html);
});

console.log(`Generados ${data.productos.length} productos.`);

Ejecutar después de cada actualización del JSON:

node scripts/generate-catalog.js
